## 🎯 5 分钟快速体验

### 步骤 1: 登录系统

1. 访问 http://kpl.17kx.net
2. 使用默认账号登录：
   - 用户名: `admin`
   - 密码: `admin888`

### 步骤 2: 查看系统监控

1. 点击左侧菜单"系统监控"
2. 查看实时的系统资源使用情况
3. 观察 CPU、内存、磁盘的使用趋势图

### 步骤 3: 体验 AI 分析

1. 点击左侧菜单"AI 分析"
2. 查看 AI 服务状态（应该显示"已启用"）
3. 在搜索框中输入钱包地址
4. 选择分析类型（建议全选）
5. 点击"开始分析"
6. 等待 10-15 秒
7. 查看详细的分析结果

### 步骤 4: 导入钱包

1. 点击左侧菜单"批量导入"
2. 输入钱包地址（每行一个）
3. 添加标签（可选）
4. 点击"开始导入"
5. 观察实时进度

### 步骤 5: 查看钱包列表

1. 点击左侧菜单"钱包列表"
2. 查看已导入的钱包
3. 点击任意钱包查看详情
4. 查看钱包的评分、标签、交易记录

## 💡 使用技巧

### 技巧 1: 优先分析高分钱包

AI 分析需要成本，建议优先分析高分钱包：

1. 在钱包列表中按评分排序
2. 选择评分 > 75 的钱包
3. 使用"高"优先级进行分析

### 技巧 2: 利用缓存节省成本

AI 分析结果会缓存 3-7 天：

- 首次分析需要等待
- 后续查看直接使用缓存
- 除非有重大变化，不需要强制重新分析

### 技巧 3: 批量导入时添加标签

导入钱包时添加标签，方便后续管理：

```
0x123... 高频交易
0x456... 大户
0x789... 潜力新星
```

### 技巧 4: 定期查看系统监控

建议每天查看一次系统监控：

- 确保系统资源充足
- 检查业务指标
- 查看 AI 使用情况

### 技巧 5: 关注 AI 使用统计

在 AI 分析页面查看使用统计：

- 今日调用次数
- 今日成本
- 剩余额度

确保不超过预算。

## 📊 数据解读

### 钱包评分

**评分范围**: 0-100

- **90-100**: 顶级交易者，强烈推荐
- **80-89**: 优秀交易者，推荐关注
- **70-79**: 良好交易者，值得关注
- **60-69**: 一般交易者
- **< 60**: 表现较差

### AI 分析结果

#### 交易风格

- **激进型**: 高频交易、高杠杆、短线操作
- **稳健型**: 中等频率、适度杠杆、中线持仓
- **保守型**: 低频交易、低杠杆、长线持仓

#### 风险等级

- **低风险**: 风险评分 < 40
- **中等风险**: 风险评分 40-70
- **高风险**: 风险评分 > 70

#### 策略有效性

- **高效**: 有效性 > 0.7
- **中等**: 有效性 0.5-0.7
- **低效**: 有效性 < 0.5

## 🔍 常见问题

### Q1: AI 分析需要多长时间？

**A**: 首次分析需要 10-15 秒，后续使用缓存立即返回。

### Q2: AI 分析的成本是多少？

**A**: 单次综合分析约 ¥0.0075，每日限制 1000 次。

### Q3: 如何提高分析准确度？

**A**: 
- 确保钱包有足够的交易历史
- 选择多种分析类型
- 定期重新分析以获取最新结果

### Q4: 系统监控显示资源使用率高怎么办？

**A**: 
- 检查是否有大量并发任务
- 考虑升级服务器配置
- 优化数据采集频率

### Q5: 如何修改 AI 配置？

**A**: 
1. 访问"系统配置"页面
2. 找到 AI 配置部分
3. 修改参数（需要管理员权限）
4. 保存并重启服务

## 🎓 进阶使用

### 使用 API

如果你熟悉编程，可以直接使用 API：

```bash
# 登录获取 token
curl -X POST http://kpl.17kx.net/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username": "admin", "password": "admin888"}'

# 分析钱包
curl -X POST http://kpl.17kx.net/api/ai/analyze \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "wallet_address": "0x...",
    "analysis_types": ["style", "strategy", "risk"]
  }'
```
